## Описание архитектуры разработанного ПО.
Система рарзработанна на языки Си, взаимодействие происзодит через консольный режим.
# Основыне модули
- queue.h - интерфейс работы с очередью
- queue.c - реализация интерфейса
- main.c - точка входа программы, обработка полученных команда
- errors.h - файл с описанием вохможных ошибок

## Количество списков
Используется количество списков равное количество приоритетов, такии образом каждый список хранит документы своего приоритета.
Каждый список представляет собой очередь реализованную по принципу FIFO, из за чего получается прямой доступ к элементам высшего приоритета.

## Счетчик операций
Общее количество обработанных заявок хранится в структуре очереди, поле total. Счетчик количества шагов обхода списка хранится внутри функции get_next_document как локальная переменная и возвращается через указатель в вызывающую сторону.

## Временная сложность NEXT
В лучшем случае O(1) в худщем O(n), где n - количество приоритетов. Происходит поиск документов с наивысшим приоритетом, каждая проверка - это сравнение указателя головы списка. Время можно улучшить, если изменить способ хранения очередей, так чтобы на 0 месте в массиве всегда находился непустая очередь.
